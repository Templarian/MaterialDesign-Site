Release Management

For a release to go out the list below needs to be tested.

  • Assign Codepoints
  • Generate @mdi/svg
  • Verify @mdi/svg works with the webfont builder
  • Generate @mdi/font
  • Update @mdi/svg and @mdi/font repos.
  • @mdi/js uses npm run build
  • @mdi/angular-material uses npm run build

FAQ

  • Q. Icon is Missing

    A. Make sure it is published and not in a draft state.

  • Q. What is a codepoint?

    A. A codepoints is a index (hex encoded) in a font. These must be unique within a single font and should not change between releases.

  • Q. How do I know what codepoint to use?

    A. Assuming you want the next in the series simply click in a blank input.

  • Q. Can I assign codepoints out of order?

    A. No, it will literally break everything.
